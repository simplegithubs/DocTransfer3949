## Prompt 7: 🛡️ Security Whitepaper Summary

> **Type:** Technical Content — Trust-building  
> **Goal:** Establish credibility with security-conscious enterprises

```
Write a 1-page security whitepaper summary for DocTransfer covering:

1. Encryption Architecture:
   - Client-side AES-256 encryption (Vault Mode)
   - Keys never stored on server — passed via URL hash fragment
   - Encryption IV stored separately from keys
   - Zero-knowledge architecture (even DocTransfer can't read files)

2. Access Control Layers:
   - Password protection with server-side hashing
   - Email-based verification (restrict by email address)
   - Link expiration (date-based or duration-based: hours/days/weeks)
   - Max view limits & burn after reading
   - Biometric lock (fingerprint & Face ID)
   - Webcam snapshot on document view
   - Device fingerprinting

3. Document Protection:
   - Dynamic watermarking (text, opacity, rotation, tiled/single)
   - DRM protection & copy prevention
   - Download restriction controls
   - Forensic watermark tracking

4. Compliance & Auditing:
   - Full audit trail with timestamps, IP, device info
   - Compliance report generation (HIPAA, GDPR, NIST alignment)
   - Digital certificates for e-signatures
   - Activity logging for every document interaction

Tone: Technical, authoritative, concise. Suitable for a CISO or DPO audience.
```
